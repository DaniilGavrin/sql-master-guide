## 🚀 О проекте

Этот репозиторий содержит исчерпывающую документацию по языку структурированных запросов **SQL**. Здесь собраны все необходимые команды, примеры использования, лучшие практики написания запросов и стратегии оптимизации.

Главная цель — предоставить ресурс, которым можно пользоваться ежедневно:
*   🌐 **Онлайн-версия**: Удобный сайт с поиском и навигацией.
*   🖨️ **PDF-шпаргалка**: Готовый файл для печати, содержащий все команды в одном месте.
*   📚 **Структурированные гайды**: От простого `SELECT` до сложных оконных функций и транзакций.

## 📖 Что внутри?

Документация разбита на логические блоки для удобного изучения:

### 🟢 Для новичков
*   **Основы SQL**: Синтаксис, типы данных, создание таблиц (`CREATE`, `ALTER`, `DROP`).
*   **Выборка данных**: `SELECT`, фильтрация (`WHERE`), сортировка (`ORDER BY`).
*   **Агрегация**: Группировка (`GROUP BY`), функции `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`.

### 🟡 Для уверенных пользователей
*   **Соединения таблиц**: `INNER JOIN`, `LEFT/RIGHT JOIN`, `FULL JOIN`, `CROSS JOIN`.
*   **Подзапросы**: Вложенные запросы, коррелирующие подзапросы.
*   **Модификация данных**: `INSERT`, `UPDATE`, `DELETE`, транзакции (`BEGIN`, `COMMIT`, `ROLLBACK`).

### 🔴 Для профи
*   **Оконные функции**: `ROW_NUMBER`, `RANK`, `DENSE_RANK`, `LEAD`, `LAG`.
*   **Оптимизация**: Индексы, планы выполнения запросов (`EXPLAIN`), оптимизация джойнов.
*   **Продвинутые объекты**: Представления (`VIEW`), хранимые процедуры, триггеры, функции.
*   **Управление доступом**: Роли, привилегии (`GRANT`, `REVOKE`).

## 🛠 Как использовать этот репозиторий

### 1. Просмотр онлайн
Просто перейдите на [сайт документации](https://daniilgavrin.github.io/sql-master-guide/), созданный через **MkDocs**. Сайт автоматически обновляется при пуше в ветку `main`.

### 2. Локальная разработка документации
Если вы хотите дополнить гайд:
```bash
# Клонирование
git clone https://github.com/DaniilGavrin/sql-master-guide.git
cd sql-master-guide

# Установка зависимостей (требуется Python)
pip install mkdocs mkdocs-material

# Запуск локального сервера
mkdocs serve
```
Откройте `http://127.0.0.1:8000` в браузере.

### 3. Печать шпаргалки
Скачайте PDF из релизов или сгенерируйте его самостоятельно из файла `to_print/sql_full_cheatsheet.md`.
