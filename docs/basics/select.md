# SELECT запросы

## Основы выборки данных

Оператор `SELECT` — это самый часто используемый оператор в SQL. Он позволяет извлекать данные из одной или нескольких таблиц.

## Синтаксис

```sql
SELECT column1, column2, ...
FROM table_name;
```

## Выбор всех столбцов

Используйте `*` для выбора всех столбцов таблицы:

```sql
SELECT * FROM users;
```

!!! warning "Внимание"
    Использование `SELECT *` не рекомендуется в продакшене, так как:
    - Возвращает больше данных, чем нужно
    - Замедляет выполнение запроса
    - Может нарушиться при изменении структуры таблицы

## Выбор конкретных столбцов

```sql
SELECT 
    first_name,
    last_name,
    email
FROM users;
```

## Псевдонимы столбцов (AS)

Можно давать столбцам временные имена с помощью `AS`:

```sql
SELECT 
    first_name AS имя,
    last_name AS фамилия,
    email AS электронная_почта
FROM users;
```

## Уникальные значения (DISTINCT)

Для получения уникальных значений используйте `DISTINCT`:

```sql
-- Получить все уникальные города
SELECT DISTINCT city FROM customers;

-- Несколько столбцов
SELECT DISTINCT country, city FROM customers;
```

## LIMIT и OFFSET

Для ограничения количества возвращаемых строк:

```sql
-- Первые 10 записей
SELECT * FROM products LIMIT 10;

-- Пропустить первые 5 и взять следующие 10
SELECT * FROM products LIMIT 10 OFFSET 5;
```

## Практические примеры

### Пример 1: Простая выборка

```sql
SELECT 
    product_name,
    price,
    category
FROM products;
```

### Пример 2: С псевдонимами и вычислениями

```sql
SELECT 
    product_name AS товар,
    price AS цена,
    price * 1.20 AS цена_с_ндс
FROM products;
```

## Частые ошибки

❌ **Неправильно:**
```sql
SELECT * FORM users;  -- Опечатка: FORM вместо FROM
```

✅ **Правильно:**
```sql
SELECT * FROM users;
```
